# 英语场景闯关小游戏 - 产品需求文档

## 1. 产品概述

一款沉浸式英语学习闯关游戏,通过模拟真实生活场景(餐厅、机场、酒店、商场、问路),让玩家在游戏互动中学习实用英语对话。每个场景都是一次探险,玩家需要完成各种任务来推进剧情,同时提升英语听说能力。

**核心价值:** 将英语学习游戏化,通过情景剧式的交互体验,让用户在实际应用场景中掌握地道英语表达。

**目标用户:** 英语初学者至中级学习者,希望提升日常英语交流能力的学习者。

## 2. 核心功能

### 2.1 场景设计

游戏包含 **5 个经典生活场景**:

1. **🍽️ 餐厅场景 (Restaurant)**
   - 场景描述:一家优雅的西餐厅,需要用英语点餐
   - 任务:选择菜品、询问菜品详情、结账买单
   - 学习重点:点餐用语、询问建议、表达喜好

2. **✈️ 机场场景 (Airport)**
   - 场景描述:国际机场出发大厅,办理登机手续
   - 任务:值机办理、托运行李、过安检、登机广播
   - 学习重点:机场常用词汇、对话表达、紧急情况处理

3. **🏨 酒店场景 (Hotel)**
   - 场景描述:星级酒店前台,办理入住和咨询服务
   - 任务:预订房间、登记入住、客房服务、投诉处理
   - 学习重点:酒店入住流程、礼貌用语、问题沟通

4. **🛍️ 商场场景 (Shopping Mall)**
   - 场景描述:大型购物中心,购物和讨价还价
   - 任务:寻找商品、询问价格、试穿衣物、付款退货
   - 学习重点:购物用语、尺寸颜色、讨价还价

5. **📍 问路场景 (Directions)**
   - 场景描述:城市街头,向路人问路
   - 任务:找人问路、理解方位描述、指路帮助他人
   - 学习重点:方位词组、交通工具、方位描述

### 2.2 游戏机制

每个场景包含 **3 个难度等级**:

- **初级**:选择题形式,4选1,提示较多
- **中级**:填空题形式,需要输入关键单词
- **高级**:情景对话,需要选择正确的完整句子

**闯关流程:**

1. 选择场景 → 进入该场景的介绍页面
2. 选择难度 → 根据当前关卡开放程度选择
3. 开始闯关 → 完成对话任务获得积分
4. 评分系统 → 正确率 >80% 过关,获得星星奖励
5. 解锁下一关 → 攒够星星解锁新场景

### 2.3 用户交互

**核心交互模式:**

- **对话选择**:点击选项完成对话
- **语音播放**:每个句子都有标准发音(使用 Web Speech API)
- **即时反馈**:正确答案显示鼓励动画,错误显示正确用法
- **积分系统**:每关最多3颗星,累积可用于解锁场景
- **进度追踪**:显示已完成关卡和未解锁内容

### 2.4 页面结构

| 页面名称 | 主要功能 |
|---------|----------|
| 首页 | 场景选择、进度展示、积分显示 |
| 场景大厅 | 5个场景入口,每个场景显示完成度 |
| 游戏页面 | 对话界面、选项、计时器、积分 |
| 结果页面 | 得分展示、星星奖励、下一关按钮 |
| 成就页面 | 收集的成就徽章展示 |

### 3.2 用户体验流程

1. **首次进入**:欢迎动画 → 快速教程 → 进入首页
2. **游戏过程**:沉浸式对话 → 即时反馈 → 趣味动画
3. **关卡结束**:成绩展示 → 星星动画 → 继续或返回

## 4. 用户界面设计

### 4.1 设计风格

**整体美学:** 现代扁平化 + 卡通插画风格,活泼友好但不失专业感

**配色方案:**

- 主色调:天蓝色 `#4F46E5`(代表信任和智慧)
- 辅助色:珊瑚橙 `#F97316`(活力和趣味)
- 背景色:浅灰白 `#F8FAFC`
- 文字色:深灰 `#1E293B`
- 成功色:翠绿 `#10B981`
- 错误色:玫红 `#EF4444`

**字体选择:**

- 标题字体:Poppins (现代感强)
- 正文字体:Inter (清晰易读)
- 对话字体:Comic Neue (友好亲切)

**按钮风格:**

- 圆角按钮(20px radius)
- 轻微阴影效果
- 悬停放大+颜色变化
- 点击反馈动效

**布局风格:**

- 卡片式布局,每个场景一张卡片
- 大图标+文字说明
- 清晰的视觉层级
- 流畅的页面切换动画

**动效设计:**

- 页面切换:淡入淡出 + 轻微位移
- 正确答案:绿色闪烁 + 星星飞散
- 错误答案:红色抖动 + 显示正确答案
- 场景解锁:礼花动画
- 进度更新:数字滚动效果

### 4.2 主要页面设计

**首页设计:**

- 顶部:用户头像 + 积分显示
- 中部:5个场景卡片横向排列
- 每个卡片:场景图标 + 名称 + 完成度 + 星星数
- 底部:成就入口按钮

**游戏页面设计:**

- 顶部:关卡标题 + 计时器 + 积分
- 中部:场景对话气泡(角色头像 + 对话内容)
- 底部:4个选项按钮
- 侧边:提示按钮(消耗积分)

**结果页面设计:**

- 中央:大号分数显示
- 星星动画:从0到实际数量依次出现
- 按钮:重试 / 下一关 / 返回首页

### 4.3 响应式设计

- **桌面端(>1024px)**:场景卡片2行3列网格
- **平板端(768-1024px)**:场景卡片2列布局
- **移动端(<768px)**:单列垂直滚动,底部固定导航

## 5. 技术实现要求

### 5.1 前端技术栈

- **框架**:React 18 (TypeScript)
- **样式**:Tailwind CSS
- **动画**:CSS Animations + Framer Motion
- **状态管理**:React Context + useReducer
- **数据存储**:LocalStorage (保存游戏进度)
- **语音合成**:Web Speech API

### 5.2 数据结构

```typescript
interface Scene {
  id: string;
  name: string;
  icon: string;
  description: string;
  levels: Level[];
  unlocked: boolean;
  stars: number;
}

interface Level {
  id: string;
  difficulty: 'easy' | 'medium' | 'hard';
  questions: Question[];
  unlocked: boolean;
  stars: number;
}

interface Question {
  id: string;
  scene: string;
  npcDialogue: string;
  options: string[];
  correctAnswer: number;
  hint?: string;
  explanation?: string;
}
```

### 5.3 性能要求

- 首屏加载时间 < 2秒
- 页面切换流畅,无白屏
- 动画帧率保持 60fps
- 移动端触控响应 < 100ms

### 5.4 无障碍要求

- 支持键盘导航
- 颜色对比度符合 WCAG 2.1 AA 标准
- 所有交互元素有适当的焦点状态
- 支持屏幕阅读器
